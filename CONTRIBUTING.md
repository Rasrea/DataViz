# 🤝 贡献指南

感谢您对本项目的兴趣！我们欢迎您的贡献，无论是修复 Bug、优化代码，还是添加新功能。请阅读以下指南，以便更顺利地贡献代码。

## 💡 如何提交 Issue
- 先 **搜索** 是否已有类似的 Issue，避免重复提交。
- 提交 Issue 时，请提供 **清晰的描述**，包括：
  - 复现步骤
  - 期望的行为
  - 实际的行为
  - 环境信息（如 Java 版本、操作系统等）

## 🛠️ 如何提交 Pull Request (PR)
1. **Fork** 本仓库并 Clone 到本地：
   ```bash
   git clone git@github.com:Rasrea/DataViz.git
   cd your-repo
   ```
2. 创建新的分支进行开发：
   ```bash
   git checkout -b feature-new-function
   ```
3. 提交代码：
   ```bash
   git add .
   git commit -m "✨ 添加新功能：支持数据导入"
   ```
4. 推送到远程仓库：
   ```bash
   git push origin feature-new-function
   ```
5. 在 GitHub 上发起 Pull Request，并描述改动内容。

## 📏 代码规范
- 使用 **Java 17+** 进行开发
- 代码风格遵循 **Google Java Style**
- 变量命名清晰，方法注释完整

## 📝 贡献类型
- 🐛 **修复 Bug**（fix: 修复某个问题）
- ✨ **新增功能**（feat: 添加新功能）
- 📖 **优化文档**（docs: 更新 README 或其他文档）

## 🎯 贡献者须知
- PR 需要经过 Code Review 后合并
- 请确保代码通过单元测试
- 遵守社区行为准则（Code of Conduct）
